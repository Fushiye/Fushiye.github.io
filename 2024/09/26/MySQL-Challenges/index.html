常见错误及注意事项：

聚合函数

  1. 聚合函数和group by是同时进行的，一边遍历所有数据进行分组，一边进行聚合函数计算
  2. 所有非聚合列必须在 GROUP BY 子句中列出，不能在 GROUP BY 中使用别名。
  3. HAVING 可以使用聚合函数，但 WHERE 不能，HAVING 必须在 GROUP BY 之后使用。
  4. 可以在 ORDER BY 中使用聚合函数。

子查询

  1. 子查询结果作为表、视图、临时表时必须取别名

  2. 子查询可以和父查询联动（相关子查询（correlated subquery）），查看错题（3）[子查询] 各部门前n高的成员

流程控制

  1. IF 语句只能在存储过程、函数、触发器和事件中使用，不能直接在普通的 SQL 查询中使用。

    普通的 SQL 查询中可以使用if(condition,truevalue,falsevalue)代替

  2. CASE 语句可以在 SQL 查询中使用，只能出现在 SELECT、WHERE、HAVING 等 SQL 子句中。

语句结构和顺序

SELECT 语句的执行顺序（在 MySQL 和 Oracle 中，SELECT 执行顺序基本相同）：
FROM -> (ON)WHERE -> GROUP BY -> HAVING -> SELECT 的字段 -> DISTINCT -> ORDER BY -> LIMIT
