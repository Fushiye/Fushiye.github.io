ES6

ES6，全称为ECMAScript 2015，是JavaScript语言的一个重要版本更新。它引入了许多新特性，旨在提高代码的可读性、简洁性和开发效率。主要是ES6的一些关键特性和改进：

变量和模板字符串

let 和 const

ES6新增了两种声明变量的方式

  • **let**：用于声明块级作用域的变量，解决了var声明变量时存在的变量提升问题。
  • **const**：用于声明常量，一旦赋值就不能再改变。对于对象和数组，虽然不能重新赋值，但可以修改其内容。

模板字符串

模板字符串是ES6引入的一种新特性，使用反引号（`）包裹的字符串可以包含嵌入表达式。它们使得字符串拼接和多行文本处理变得更加简洁和直观。以下是模板字符串的主要特点和用法。

主要特点

  1. 嵌入表达式：使用 ${expression} 语法在字符串中嵌入JavaScript表达式。

  2. 多行字符串：可以直接在模板字符串中写多行文本，而不需要使用换行符 \n 或者字符串连接操作符 +。

  3. 简化字符串拼接：不需要再使用字符串连接操作符 + 来拼接变量和字符串。

基本用法

1. 嵌入表达式

可以在模板字符串中嵌入任何有效的JavaScript表达式，包括变量、函数调用等。

2. 嵌入函数调用

也可以在模板字符串中嵌入函数调用的结果。

3. 多行字符串

模板字符串支持直接编写多行文本，无需使用换行符 \n 或者字符串连接操作符 +。

箭头函数

基本和Java中的lambda表达式，只是->换成了=>

箭头函数是ES6引入的一种新的函数定义方式，它提供了更加简洁的语法，并且在处理 this 关键字时有一些不同的行为(箭头函数没有自己的this，它的this是外层环境中的this) 。箭头函数非常适合用于编写短小、简单的函数表达式。

下是箭头函数的主要特点和优势：

  1. 简洁的语法：

    • 单行表达式可以直接返回结果，无需 return 关键字。
    • 参数较少时可以省略括号。

  2. 固定的 this 上下文：

    • 箭头函数内部的 this 继承自外层作用域，不会随着函数调用方式而改变。

  3. 不支持 arguments 对象：

    • 可以通过剩余参数（rest parameters）替代 arguments 对象。

  4. 不能用作构造函数：

    • 不能使用 new 关键字来实例化对象。

箭头函数与普通函数的区别

  1. this 关键字的行为

    • 普通函数：this 的值取决于函数的调用方式，通常指向调用该函数的对象。
  • 箭头函数：this 的值继承自外层作用域，在箭头函数中 this 不会改变。

  1. 不能作为构造函数：箭头函数不能用作构造函数，即不能使用 new 关键字来实例化对象。

  2. 没有 arguments 对象：箭头函数内部没有自己的 arguments 对象，但可以通过剩余参数（rest parameters）来实现类似的功能。

实际应用场景

1. 回调函数

箭头函数非常适合用于简写回调函数，特别是在数组操作方法（如 map、filter、reduce 等）中。

2. 事件处理

在处理DOM事件时，箭头函数可以帮助避免 this 关键字的混淆问题。

3. 定时器和异步代码

在定时器或异步代码中，箭头函数可以确保 this 的正确性。

模块化处理

ES6（ECMAScript 2015）引入了原生的模块化支持，允许开发者将代码分割成多个文件，并通过 import 和 export 关键字来导入和导出模块。这种模块化机制提供了更好的代码组织、可维护性和复用性。

在 ES6 中，每个 JavaScript 文件被视为一个独立的模块。模块可以包含变量、函数、类等，并且可以通过 export 关键字将其公开给其他模块使用。其他模块则可以通过 import 关键字来导入这些公开的内容。

导出模块（Export）

1. 命名导出（Named Export）

你可以从一个模块中导出多个变量、函数或类，并为它们指定名称。这被称为命名导出。

2. 默认导出（Default Export）

每个模块只能有一个默认导出。默认导出不需要指定名称，通常用于导出单个主要功能或对象。

1. 导入命名导出

动态导入（Dynamic Import）

ES6 还支持动态导入，即在运行时按需加载模块。这对于优化性能非常有用，尤其是在需要延迟加载某些模块的情况下。

模块的作用域和隔离

每个模块都有自己的作用域，这意味着在一个模块中声明的变量、函数或类不会影响其他模块。只有通过 export 导出的内容才能被其他模块访问。

实际应用场景

1. 代码组织和复用

模块化使得代码更加有条理，便于管理和维护。不同的功能可以放在不同的模块中，避免了全局命名空间的污染。

2. 第三方库的集成

许多现代前端框架和库都采用模块化设计，使得你可以轻松地集成第三方库。

3. 代码分割和懒加载

通过动态导入，可以在需要时才加载模块，从而减少初始加载时间并提高应用性能。

总结

ES6 的模块化机制提供了一种强大的方式来组织和管理代码，使得开发大型应用程序变得更加容易。以下是其主要特点和优势：

  1. 清晰的作用域：每个模块都有独立的作用域，避免了全局命名空间的污染。
  2. 命名导出和默认导出：支持多种导出方式，满足不同的需求。
  3. 灵活的导入方式：可以根据需要选择导入特定内容或整个模块。
  4. 动态导入：支持按需加载模块，有助于优化性能。
  5. 代码复用和维护：模块化使代码更易于复用和维护，提升了开发效率。

通过合理使用 ES6 的模块化特性，开发者可以构建更加健壮、高效和可维护的应用程序。无论是在小型项目还是大型系统中，模块化都是提升代码质量和开发体验的重要工具。

Node.js&NPM

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行时，它使得开发者可以使用 JavaScript 编写服务器端应用程序。Node.js 的设计初衷是为了提供一种高效的方式来处理 I/O 密集型任务，如文件系统操作、网络请求等。有了Node.js后，JavaScript就从一个普通的前端语言转变成可以操作服务器文件的全栈语言。

主要特点：

  1. 事件驱动和非阻塞 I/O：Node.js 使用事件循环（Event Loop）来处理异步操作，这使得它可以非常高效地处理并发请求。
  2. 单线程模型：尽管是单线程，但由于其异步和非阻塞特性，Node.js 仍然能够处理大量并发连接。
  3. 跨平台支持：可以在 Windows、macOS 和 Linux 等多种操作系统上运行。
  4. 丰富的生态系统：通过 npm（Node Package Manager），Node.js 拥有庞大的第三方库和工具生态系统。

安装参考：菜鸟教程 - node.js的安装

单文件组件

在vue3框架中，通过.vue文件来管理组件（组件：前端页面中，多个页面可能存在相同的模块，可以将这些相同的代码提取成一个单独的组件，在不同文件引入即可）。单文件组件（SFC） 是 Vue.js 中一个非常重要的概念，它允许你在一个单独的文件中定义组件的模板、逻辑和样式。这种模式使得代码更加模块化、可维护，并且提高了开发效率。Vue 的单文件组件使用 .vue 文件扩展名。每个 .vue 文件通常包含三个主要部分：<template>、<script> 和 <style>，它们分别用于定义组件的模板、逻辑和样式。

单文件组件的组成部分

  1. <template> 部分

    • 作用：定义组件的 HTML 结构。内部只能有一个一级子标签。

    • 语法：使用标准的 HTML 语法，并支持 Vue 的指令（如 v-bind, v-on 等）和插值表达式（如 {{ }}）。

  2. <script> 部分

    • 作用：定义组件的 JavaScript 逻辑。
    • 语法：通常使用 ES6 模块化的 export default 语法来导出一个对象，该对象包含了组件的各种选项，如 data、methods、computed 等。

  3. <style>部分

    • 作用：定义组件的 CSS 样式。
    • 语法：可以使用标准的 CSS 语法，也支持预处理器（如 Sass、Less）。通过添加 scoped 属性，可以确保样式仅应用于当前组件，避免全局样式冲突。

工程文件间的逻辑关系

说明: 两个主要文件分别是 HTML 入口文件 index.html和 JavaScript 主入口文件main.js，它们共同构成了一个 Vue 应用的基础结构。index.html是一个标准的 HTML 文件，作为整个应用的入口点。它定义了页面的基本结构，并包含了一些必要的元数据和脚本引用。main.js是 Vue 应用的主要入口文件，负责创建和初始化 Vue 应用实例，并将其挂载到 HTML 文件中的指定 DOM 元素上。根组件可以再导入其他组件。

工作流程:

  1. **浏览器加载 index.html**：
    • 浏览器首先加载并解析 index.html 文件，识别出其中的 <script> 标签，该标签指向 /src/main.js。
  2. 执行 main.js 文件：
    • 当浏览器遇到 <script> 标签时，它会请求并加载 /src/main.js 文件。
    • 在 main.js 文件中，首先导入必要的模块和组件。
    • 使用 createApp 方法创建一个新的 Vue 应用实例，并将 App.vue 作为根组件。
    • 最后，通过 .mount('#app') 将这个应用实例挂载到 HTML 文件中 ID 为 app 的 div 元素上。
  3. Vue 应用渲染内容：
    • Vue 应用启动后，它会根据 App.vue 组件的内容生成相应的 DOM 结构，并将其插入到 index.html 中的 div#app 元素内。
    • 用户可以在浏览器中看到最终渲染后的页面。

路由守卫

Vue Router 提供了路由守卫（Navigation Guards）机制，允许你在导航过程中执行一些逻辑，例如权限验证、页面加载前的数据预取等。路由守卫可以帮助你控制导航行为，确保用户只能访问他们有权限的页面。

路由守卫的类型

Vue Router 提供了三种类型的路由守卫：

  1. 全局前置守卫（Global Before Guards）：全局前置守卫在每次路由切换之前都会被调用。
  2. 路由独享守卫（Per-Route Guards）：可以在定义路由时为每个路由单独添加守卫。这些守卫只会在进入该特定路由时触发。
  3. 组件内守卫（In-Component Guards）：可以在组件内部定义导航守卫。这些守卫只会影响包含它们的组件。

除了上述三种主要类型的守卫，Vue Router 还提供了其他几种辅助性的守卫和钩子

  • 全局解析守卫：在导航被确认之前处理异步操作。

  • 后置钩子：在导航完成后执行的操作

Promise简介

回调函数：

回调函数（Callback Function）是 JavaScript 中处理异步操作和事件驱动编程的一种常见模式。回调函数是一个作为参数传递给另一个函数的函数，并在某个特定事件或条件发生时被调用。回调函数通常用于处理异步操作、事件监听等场景。在 JavaScript 中，许多异步操作（如网络请求、文件读取、定时器等）都使用回调函数来处理操作完成后的逻辑。

示例：定时器

这里 setTimeout 是一个内置的异步函数，它接受两个参数：一个回调函数和一个延迟时间（毫秒）。回调函数将在指定的延迟时间后执行。

当多个异步操作需要按顺序执行时，可能会出现嵌套的回调函数，这种现象被称为“回调地狱”。回调地狱不仅使得代码难以阅读和维护，还容易引发错误。为了解决回调地狱问题，JavaScript 提供了多种解决方案，包括 Promise、async/await 等。

Promise 是一种用于处理异步操作的对象。它表示一个异步操作的最终完成（或失败）及其结果值。Promise 提供了一种更清晰的方式来处理异步代码，避免了回调地狱（Callback Hell）的问题。

**Promise 的状态: **

一个 Promise 对象代表一个异步操作，有三种状态：

  1. Pending（进行中）：初始状态，既不是成功，也不是失败。
  2. Fulfilled（已成功）：操作成功完成。
  3. Rejected（已失败）：操作失败。

只有异步操作的结果可以决定当前是哪一种状态，任何其他操作都无法改变整个状态，一旦 Promise 被解决（fulfilled 或 rejected），它的状态就不可再改变，并且结果值也会被固定下来，此后任何时候都可以得到这个结果。

方法2：预检请求：浏览器向跨域发送请求前先向跨域发送预检请求，跨域回复安全确认信息。预检请求抗议设置时效，在此时效内仅第一次请求前发送。跨域的服务器必须有跨域过滤器，用于判断请求是否是预检请求，便于响应允许跨域信息。
