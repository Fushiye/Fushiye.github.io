微服务是一种软件架构风格，它将一个完整的应用程序拆分为一组小型、独立部署的服务，每个服务专注于完成特定的业务功能，并通过轻量级的通信机制（如 HTTP API）进行交互。这些服务可以独立开发、测试、部署和扩展，从而提高系统的灵活性、可维护性和可伸缩性，特别适合复杂的大型应用开发。Cloud系列主要是学习学习微服务和SpringCloud过程中记录的笔记，本文是第一篇，主要涉及微服务项目开发规范以及学习过程中遇到问题及重要知识点总结。

基本说明

SpringCloud简介

Spring Cloud 是一个用于构建分布式系统和微服务架构的开源框架，基于 Spring Boot 开发。它提供了一套工具和组件，帮助开发者快速实现微服务之间的通信、配置管理、负载均衡、服务注册与发现等功能。适合希望采用微服务架构的企业级应用开发。

关键点：

  1. 微服务支持：简化微服务开发，解决服务间通信、容错、监控等问题。微服务是一种软件架构风格，它将一个完整的应用程序拆分为一组小型、独立、自治的服务，每个服务专注于完成特定的业务功能，并通过轻量级的通信机制（如 HTTP/REST API、消息队列等）进行交互。
  2. 核心模块：
    • Spring Cloud Netflix（如 Eureka、Ribbon、Hystrix）：服务注册与发现、负载均衡、断路器。
    • Spring Cloud Config：集中化配置管理。
    • Spring Cloud Gateway/Zuul：API 网关。
    • Spring Cloud Stream/Bus：消息驱动与事件总线。
  3. 适用场景：适用于需要构建高可用、可扩展的分布式系统的项目。
  4. 优势：整合性强，生态丰富，与 Spring Boot 高度集成，降低微服务开发复杂度。

父工程准备

包含的核心依赖及其作用：

  • Spring Boot：引入 spring-boot-starter-parent，版本为 3.2.0，作为基础框架。
  • Spring Cloud：引入 spring-cloud-dependencies，版本为 2023.0.0，支持分布式系统开发。
  • Spring Cloud Alibaba：引入 spring-cloud-alibaba-dependencies，版本为 2022.0.0.0-RC2，集成阿里巴巴的中间件组件。
  • MyBatis：通过 mybatis-spring-boot-starter 集成 MyBatis 框架，版本为 3.0.2。

  • 通用 Mapper：引入 tk.mybatis.mapper，版本为 4.2.3，简化 MyBatis 的 CRUD 操作。
  • FastJSON2：集成阿里巴巴的 JSON 序列化工具，版本为 2.0.40。
  • Swagger3：通过 springdoc-openapi-starter-webmvc-ui 提供 API 文档生成功能，版本为 2.2.0。
  • Hutool：引入 hutool-all 工具包，版本为 5.8.22，提供丰富的工具类。
  • Lombok：集成 Lombok 工具，版本为 1.18.26，减少样板代码。
  • JUnit 测试：引入 spring-boot-starter-test，版本为 3.1.5，用于单元测试。

业务类中直接将generator中生成的xml、entities、mapper文件直接复制过来，mapper.xml文件需要置于resources目录下。一个好的工程应当做到如下优化：

  • 如果使用swagger，需要编写配置类，同时为实体类和controller添加如@Tag，@Schema，@Operation等注解说明
  • 为了避免直接暴露领域模型或数据库实体类，降低系统各层之间的耦合度，防止敏感信息被意外传递到客户端，只传输必要的数据。需要为实体类设计一个与前端交互的DTO类
  • 为了方便前端读取数据，应当设计一个全局统一返回格式类详情，一般包括：
    • code：状态值，建议和message一起使用枚举类（包含举值、构造方法、遍历 详情）表示。
    • message：本次接口调用结果描述
    • data：本次返回的数据
    • timestamp：可能有时间戳，接口的调用时间
    • 操作成功函数、其他操作结果函数
  • 为了减少try-catch等代码，规范返回数据，应当有全局异常接入返回的标准格式.

全局异常处理类设计

注释说明：

  1. **@Slf4j**：Lombok 提供的注解，用于自动生成 log 对象，简化日志记录代码。
  2. **@RestControllerAdvice**：Spring 提供的注解，声明该类是一个全局异常处理控制器，专门用于处理 RESTful 接口中的异常。
  3. **@ExceptionHandler(RuntimeException.class)**：指定该方法用于捕获 RuntimeException 类型及其子类的异常。
  4. **@ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)**：设置当该方法被触发时，HTTP 响应的状态码为 500（内部服务器错误）。
  5. **log.error**：记录异常的详细信息，包括异常消息和堆栈跟踪，便于后续排查问题。
  6. **ResultData.fail**：调用 ResultData 的 fail 方法，返回封装后的错误结果，包含错误码和错误消息。

RestTemplate

官网介绍

RestTemplate 是 Spring 框架提供的一个用于与 RESTful 服务进行交互的客户端工具类。RestTemplate 是一个同步的 HTTP 客户端，用于发送 HTTP 请求（如 GET、POST、PUT、DELETE 等）并处理响应。在微服务架构中，服务间通信常通过 RestTemplate 实现。它可以：

  • 调用外部 RESTful API 或微服务接口。
  • 支持多种数据格式（如 JSON、XML）的请求和响应转换。
  • 提供便捷的方法封装 HTTP 请求逻辑。

使用restTemplate访问restful接口非常的简单粗暴。(url, requestMap, ResponseBean.class)这三个参数分别代表 REST请求地址、请求参数、HTTP响应转换被转换成的对象类型，常用API如下：

方法名称 功能描述 返回值类型
exchange() 发送任意类型的 HTTP 请求（GET、POST、PUT、DELETE 等），并允许自定义请求头和请求体。 ResponseEntity<T>
execute() 执行 HTTP 请求，并允许通过回调函数处理响应内容，适合更复杂的场景。 用户定义的类型
headForHeaders() 发送 HEAD 请求，用于获取目标资源的响应头信息。 HttpHeaders
put() 发送 PUT 请求，通常用于更新资源，不返回响应体。 void
delete() 发送 DELETE 请求，用于删除资源，不返回响应体。 void
optionsForAllow() 发送 OPTIONS 请求，用于获取目标资源支持的 HTTP 方法列表。 Set<HttpMethod>

  • *Entity()：返回结果为ResponseEntity对象，包含响应体中一些重要信息如响应头、状态码、响应体等。
  • *Object()：返回结果为响应体中数据转化成的对象，可以理解为JSON
  • 上述为GET请求方法对应API，POST请求方法对应API只需要将get换为post即可。

工程重构

在各个微服务项目中可能存在重复的代码，如entities、全局结果类、全局异常类等，这些代码可以提取到同一个模块中，此模块对外暴露通用的组件/API/接口/工具类等，供其他服务调用。此模块无需运行，因此不是一个独立的服务，也无需yaml配置文件，相当于一个工具模块。

对工具模块执行（生命周期中的）maven clean+maven install操作，清除其他模块的重复部分，然后在其他服务模块的POM文件的dependencys中引入当前工具模块。

Caused by: java.net.UnknownHostException: cloud-payment-service

【问题原因】

consul原生支持负载均衡，默认每个微服务都是集群状态，RestTemplate发送请求时会不清楚具体的对象。

【解决方案】

Openfeign

A component required a bean of type ‘com.fu.apis.PayFeignApi’ that could not be found. Action: Consider defining a bean of type ‘com.fu.apis.PayFeignApi’ in your configuration.

【问题概述】：

服务调用者启动时出现上述报错，无法找到对应API

【产生原因】：
@EnableFeignClients 注解的作用是启用 Feign 客户端功能，并扫描指定范围内的接口以将其注册为 Spring 管理的 Bean。如果不指定 basePackages 参数，默认情况下，Spring Boot 会基于主应用程序类（即带有 @SpringBootApplication 注解的类）所在的包及其子包进行扫描。如果 Feign 接口位于其他模块或不在主应用程序类的包结构中，则不会被自动扫描到。

【解决方案】：
使用 @EnableFeignClients(basePackages = "com.fu.apis") 明确指定 Feign 接口所在的包路径。

知识点总结

loadbalancer

依赖spring-cloud-starter-consul-discovery中实际已经包含了spring-cloud-starter-loadbalancer，且如果consul默认情况下没有使用注解@LoadBalanced开启负载均衡会无法使用，所以使用consul时此环节无需任何操作即可自动实现负载均衡！

OpenFeign

接口提取服务提供者的对外暴露方法时，最重要的是访问的URL必须和服务提供者的Controller中的一致，抽象方法名可以不一致，但为了方便查看，一般取同名。同时，方法中从配置消息中读取的参数可以省略。
