Lombok 是一个 Java 开发工具库，旨在通过注解的方式减少样板代码（Boilerplate Code）的编写。它通过在编译时自动生成常见的代码逻辑（如 getter、setter、构造函数、toString 方法、日志等），从而简化开发流程并提高代码的可读性。

介绍

Lombok 是一款 Java 开发的实用工具库，主要用于简化 Java 代码的 boilerplate（模板化代码）。它通过注解的方式，在编译时自动生成常见的代码逻辑，如 getter、setter、构造方法等。

核心功能：

  1. 减少冗余代码：自动为类生成 getter、setter、toString 等方法。
  2. 简化构造器：支持一键生成无参构造器、全参构造器或指定字段的构造器。
  3. 日志支持：通过注解快速集成常用的日志框架。
  4. 提升可读性：使代码更加简洁和易于维护。

适合需要快速开发且希望减少手动编写样板代码的场景。

  • 如果使用 IntelliJ IDEA，请安装 Lombok 插件（File -> Settings -> Plugins 搜索 “Lombok” 并安装）。
  • 安装后，重启 IDE 并启用注解处理（Settings -> Build, Execution, Deployment -> Compiler -> Annotation Processors，勾选 “Enable annotation processing”）。

2. 常用注解及其功能

(1) @Getter 和 @Setter

生成类字段的 getter 和 setter 方法。

(2) @ToString

生成 toString() 方法，默认包含所有字段。

(3) @EqualsAndHashCode

生成 equals() 和 hashCode() 方法。

(4) @Data

综合注解，包含 @Getter、@Setter、@ToString、@EqualsAndHashCode 和 @RequiredArgsConstructor。

(5) @NoArgsConstructor, @AllArgsConstructor, @RequiredArgsConstructor

生成构造方法：

  • @NoArgsConstructor：无参构造方法。
  • @AllArgsConstructor：全参构造方法。
  • @RequiredArgsConstructor：为 final 字段或带有 @NonNull 注解的字段生成构造方法。

(6) @Slf4j 或 @Log

快速集成日志功能。

3. 常见的相关配置

日志级别配置

Lombok的日志注解（如@Slf4j）会生成日志记录器，因此可以通过配置日志级别来控制日志输出。

  • root: 全局日志级别。
  • com.example: 替换为你的项目包名，用于设置特定包的日志级别。
