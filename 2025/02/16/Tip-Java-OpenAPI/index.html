Tip-Java-OpenAPI

发表于 2025-02-16 更新于 2025-03-25 分类于 Java ， 框架与工具 本文字数： 2.2k 阅读时长 ≈ 7 分钟
在当前前后端分离的软件开发过程中，API文档的生成是每个开发者都无法回避的重要任务。而SpringDoc作为一款强大的开源工具，能够为基于Spring Boot的应用程序自动生成RESTful API文档，极大地简化了这一过程。本文将介绍SpringDoc的核心功能与使用方法，从基础集成到高级定制，一步步实现快速上手并优化API文档的生成。

简介

1. 什么是SpringDoc？

SpringDoc 是一个用于在基于 Spring Boot 的应用程序中生成 OpenAPI 文档的库。它能够自动扫描项目中的 RESTful API，并生成交互式的 API 文档，通常以 Swagger UI (或 ReDoc )的形式展示。

Swagger官网：https://swagger.io/

SpringFox项目：https://github.com/springfox/springfox

SpringDoc项目：https://github.com/springdoc/springdoc-openapi

SpringDoc文档：https://springdoc.org/

2. 核心概念

  • OpenAPI：一种标准化的规范，用于描述和定义 RESTful API 的结构，包括路径、参数、请求体、响应等。
  • Swagger：一个工具集，支持 OpenAPI 规范，提供 API 文档生成、测试等功能。
  • SpringDoc：专注于与 Spring 生态系统集成，简化了 OpenAPI 文档的生成过程。

3. 为什么使用SpringDoc？

  • 自动生成 API 文档，减少手动维护的工作量。
  • 提供交互式文档界面（如 Swagger UI），方便开发者和测试人员调试 API。
  • 支持多种扩展功能，例如安全认证、分组文档等。

4. Springfox的Swagger和SpringDOC对比

Springfox 是早期用于在 Spring 项目中集成 Swagger 的常用工具，但它存在一些局限性(主要是不支持SpringBoot3.x和JDK17+)，导致现在更推荐使用 SpringDoc。以下是主要原因：

  1. 维护状态：Springfox 更新缓慢，对新版本的 Spring 和 Spring Boot 支持不足，而 SpringDoc 积极维护，兼容性更好。
  2. 复杂性：Springfox 配置较为繁琐，需要手动处理大量细节，而 SpringDoc 更加轻量级，开箱即用。
  3. 性能问题：Springfox 在大型项目中可能导致启动时间过长或内存占用过高，SpringDoc 性能更优。
  4. 社区支持：SpringDoc 的社区活跃度更高，文档和示例丰富，更容易上手。

综上，SpringDoc 更适合现代开发需求，尤其是在使用较新版本的 Spring Boot 时。

依赖、配置和注解

相关依赖介绍

依赖名称 功能描述 适用场景 是否包含UI支持
springdoc-openapi-starter-webmvc-api 提供基于 Spring Web MVC 的 API 文档生成功能，生成符合 OpenAPI 3 规范的文档。 适用于需要生成 API 文档但不需要可视化界面的传统 Web 应用程序。 否
springdoc-openapi-starter-webmvc-ui 在提供 API 文档生成功能的同时，集成了 Swagger UI，允许通过浏览器查看和测试 API。 适用于需要可视化 API 文档的项目，方便开发者、测试人员和前端工程师快速理解和测试 API。 是
springdoc-openapi-starter-webflux-api 提供基于 Spring WebFlux 的响应式应用程序的 API 文档生成功能。 适用于使用 Spring WebFlux 构建的响应式应用程序。 否
springdoc-openapi-starter-webflux-ui 在提供 API 文档生成功能的同时，为 Spring WebFlux 应用程序集成了 Swagger UI。 适用于需要可视化 API 文档的响应式应用程序。 是
springdoc-openapi-security 提供与安全相关的功能，如 OAuth2 和 JWT 认证，确保 API 文档的安全性。 适用于需要对 API 进行身份验证和授权的项目。 否
springdoc-openapi-data-rest 为 Spring Data REST 提供 OpenAPI 文档生成支持，自动扫描并生成基于 Spring Data REST 的 API 文档。 适用于使用 Spring Data REST 构建的应用程序。 否
springdoc-openapi-jaxrs 为基于 JAX-RS 的应用程序提供 OpenAPI 文档生成支持。 适用于使用 JAX-RS 构建的 RESTful 服务。 否
springdoc-openapi-kotlin 为 Kotlin 语言提供更好的支持，确保 Kotlin 特有的类型和语法能够在生成的文档中正确显示。 适用于使用 Kotlin 编写的 Spring Boot 应用程序。 否
springdoc-openapi-maven-plugin 提供 Maven 插件支持，可以在构建过程中生成静态的 OpenAPI 文档文件（如 JSON 或 YAML 格式）。 适用于需要在构建时生成静态文档的项目。 否

相关配置介绍

1. 基础配置

配置项 功能描述
springdoc.api-docs.path 设置 API 文档的基础路径，默认为 /v3/api-docs。例如：/api/v3/api-docs。
springdoc.swagger-ui.path 设置 Swagger UI 的访问路径，默认为 /swagger-ui.html。例如：/swagger-ui/index.html。

2. 分组配置

配置项 功能描述
springdoc.group-configs[0].group 定义分组名称，用于将 API 按照逻辑分组显示。
springdoc.group-configs[0].paths-to-match 指定该分组匹配的路径规则（正则表达式）。
springdoc.group-configs[0].packages-to-scan 指定该分组扫描的包名。

3. 安全配置

配置项 功能描述
springdoc.security.enabled 是否启用安全配置，默认为 true。
springdoc.security.schemes[0].name 定义安全方案的名称，例如 OAuth2 或 JWT。
springdoc.security.schemes[0].type 安全方案类型，例如 oauth2、apiKey 等。
springdoc.security.schemes[0].in 指定认证信息的位置，例如 header 或 query。
springdoc.security.schemes[0].flows.authorizationUrl 对于 OAuth2，指定授权 URL。

4. 全局信息配置

配置项 功能描述
springdoc.info.title 设置 API 文档的标题。
springdoc.info.description 设置 API 文档的描述信息。
springdoc.info.version 设置 API 文档的版本号。
springdoc.info.contact.name 设置联系人姓名。
springdoc.info.contact.url 设置联系人的 URL 地址。
springdoc.info.contact.email 设置联系人邮箱地址。
springdoc.info.license.name 设置许可证名称。
springdoc.info.license.url 设置许可证 URL 地址。

5. 扩展配置

配置项 功能描述
springdoc.use-management-port 是否在管理端口上暴露 API 文档，默认为 false。
springdoc.expand-request-body 是否默认展开请求体，默认为 false。
springdoc.expand-responses 是否默认展开响应体，默认为 false。
springdoc.try-it-out-enabled 是否启用“Try it out”功能，默认为 true。
springdoc.default-consumes-media-type 设置默认的媒体类型（如 application/json）。
springdoc.default-produces-media-type 设置默认的响应媒体类型（如 application/json）。

6. 扫描配置

配置项 功能描述
springdoc.packages-to-scan 指定需要扫描的包名，默认扫描所有包。
springdoc.paths-to-match 指定需要匹配的路径规则（正则表达式）。
springdoc.paths-to-exclude 指定需要排除的路径规则（正则表达式）。

7. 插件配置

配置项 功能描述
springdoc.show-actuator 是否显示 Actuator 的端点，默认为 false。
springdoc.cache.disabled 是否禁用缓存，默认为 false。
springdoc.skip-undocumented-models 是否跳过未文档化的模型，默认为 false。
